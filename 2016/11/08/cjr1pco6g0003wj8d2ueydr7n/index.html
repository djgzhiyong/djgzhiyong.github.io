Django学习02---视图模板

发表于 2016-11-08 | 更新于 2019-01-08

基于 python 2.7 和 django 1.10

上回的django hello world 我们对 django 的简洁快速有了初步的认识
这回我们利用 django 的视图模板创建我们的第一个html网页

依旧使用 Atom 打开我们上回第一次创建的 myproject 工程

1.在views.py中创建一个视图函数 index

关于 template_name 视图模板名称 firstapp/index.html 需要在我们的应用 firstapp 的 templates 目录中创建之（其实也可以直接在templates目录中创建index.html。增加 firstapp 应用名称目录的原因是，如果同时存在多个应用及多个同名的模板文件时，将出现模板文件名称冲突）

2.在 firstapp 中添加模板文件 index.html

我们依次在 firstapp 应用目录下添加 templates、firstapp目录和 index.html文件。完成后如下

3.配置index视图的映射uri

上回我们直接在 myproject项目的 urls.py 文件中添加了我们第一个 hello world 视图函数的 uri 映射，这样应用（firstapp）和项目(myproject)的uri映射变得混乱。不利于应用的项目之间的解耦。
于是我们需要移除 myproject urls.py 中的视图配置，转而在我们视图所在的应用目录下创建创建视图函数和 uri 之间的映射。
现在我们在应用 firstapp 目录下创建视图映射 urls.py 文件，用来映射视图和uri之间的关系

关于视图映射的匹配，我们在项目 myproject 的 urls.py 文件中通过正则表达式配置了任意字符开头的 uri 即进入应用 firstapp 的 urls.py 文件中查找。
同时在 firstapp 的 urls.py文件中配置了空 uri 映射 views.helloworld 视图函数，index 字符开始和结尾隐射 views.index 函数

4.启动浏览器检查配置是否正确

完成了以上的配置我们就可以在浏览器中查看配置的内容是否正确，和上次内容一样，我们启动开发服务器—控制台进入 myproject 目录下运行 python manage.py runserver 开启服务器，浏览器输入 http://127.0.0.1:8000/ 访问上次内容的 helloworld 视图，http://127.0.0.1:8000/index 访问我们新添加的 index 模板视图

helloworld 视图一切正常

然而新添加的 index 模板视图却……

我们悲伤的看到了 django 有史以来给我们的第一个错误页面，这个错误告诉我们应用 firstapp 下的视图模板 index.html 没有被找到。可是我们明明添加了，这不完全就是坑爹吗…..。作为初学者的我感觉很崩溃，google 、百度 、外加查阅 django 的官方文档之后才知道原来是我们的应用没有被安装到我们的项目 myproject 中，我们只是在项目的 urls.py 中指明了应用的视图映射而已，所以模板文件没有被找到

5.安装应用到项目

打开项目的 setting.py 文件，一个 django 项目的整体配置位于此文件中，我们找到 INSTALLED_APPS 这一项配置，将我们的应用安装到项目中

安装完毕之后，我们再次访问 http://127.0.0.1:8000/index 查看是否成功

终于成功访问到了我们第一个视图模板 index.html 渲染出来的页面

通过应用的安装我们初探了到应用和项目之间微妙的关键，一个项目负责所有应用的管理、配置，每个应用功能独立，和项目之前就像一个插座，可快速复用、随时插拔而这也紧紧是 django 强大的另一个点….

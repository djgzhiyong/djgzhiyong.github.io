django学习04---Model

发表于 2016-11-21 | 更新于 2019-01-08

基于 python 2.7 和 django 1.10

本节来我们开始学习 django 的模型，在前3节的基础上，学习了视图函数，基本的URL配置，视图模板，对 django 有了初步的认识，现在再学习下模型，后面结合数据库一起使用。最后来写个 blog 项目练习练习
废话不说了，开始学习吧…

1.创建第一个用户模型

继续用 atom 打开 myproject ，在 firstapp models.py 文件中创建第一个模型，由名字应该得知 django 的模型配置应该都在此文件中。在这个文件里面加入我的第一个用户模型 UserInfo

在 models.py 中我们增加了第一个用户模型，其中有3个字段，对应数据库中的字段如下


username = models.CharField(max_length=20) — username varchar(20)
password = models.CharField(max_length=20) — password varchar(20)
age = models.IntegerField() — age integer

2.使用数据库

django 支持常用的数据库Mysql、Oracle、MongoDB、Sqlite…。并且 django 环境默认搭载了一个 sqlite 数据库供开发使用，超级方便。来打开 myproject 的 settings.py 文件，定位到 DATABASES 节点看到 django 在创建项目时生成了 sqlite 数据库。真心赞

数据库配置我就先跳过，等到到时候写blog项目的时候再学习，现在就使用 django 默认创建的 sqlite 数据库来学习

3.同步数据库

创建好了 UserInfo 模型现在需要将这个模型映射成一张表吧，把模型的属性映射成表中的字段吧，这是基本的ORM模式，django 也不例外。确定了以上工作，来开始同步模型和数据库吧。

打开熟悉的命令行，切换到 myproject 目录下，依次执行 python manage.py makemigrations 命令来生成数据库迁移文件，执行完毕之后，发现 UserInfo 模型所在的 firstapp 应用的 migrations 目录下悄然的发送了变化，django 为他创建了一个数据库迁移文件。
然后我再次执行 python manage.py migrate 同步模型到数据库

OK，顺利的完成了以上两步之后，我们的模型就和数据库来了一次同步的操作，模型的变化被同步到数据库中，通过数据库可视化工具来查看 django 为我们做了哪些工作…

django 不负众望的为我们将 UserInfo 模型同步到了数据库中…
